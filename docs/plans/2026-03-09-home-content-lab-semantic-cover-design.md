# Home Content Lab Semantic Cover Design

## Goal
让 `home-content-lab.html` 里的图片卡，不再只是统一风格的渐变封面，而是能在 1 秒内直接传达对应内容：运营精选、KOL 喊单、Apple Pay 买 USDT、AI 信号、宏观突发、热门叙事、专题内容、理财机会。

## Problem
当前版本已经有卡片系统感，但最大问题是“图和内容关联度不够”。用户看到的是一套抽象气氛图，而不是内容本身。

## Confirmed Direction
采用三种图像语言混合，但按内容类型分配：

1. **具象场景化（主）**
   - 用更像真实产品/真实媒体/真实社媒截图的结构，让用户马上认出主题。
2. **媒体封面化（辅）**
   - 对宏观事件、专题卡、叙事卡，用更像新闻封面/专题头图的构图。
3. **信息图表化（局部）**
   - 只给 AI / Earn / Macro 这些天然依赖数据判断的卡片。

## Card-by-card Design

### 1. Ops / 编辑精选
问题：现在更像一张抽象科技渐变。
方案：改成更像“编辑精选封面 + 置顶快讯模块”。
- 顶部有 digest / focus 模块条
- 中部有 ticker / summary lines
- 右上角保留 small utility chip
- 构图目标：像运营团队做好的“今日重点市场摘要”截图

### 2. KOL Radar
问题：只是名字云，不够像真正的社媒热榜。
方案：改成“社媒热榜墙”。
- 左侧为榜单/帖子列
- 右侧为热度标签、头像气泡、mention 数
- `@name` 仍保留，但不只是漂浮字，要更像 feed / trend list
- 构图目标：一眼认出“哪些 KOL 在喊单、这个币在被刷屏”

### 3. Apple Pay → USDT
问题：已有支付语义，但还不够像真实支付流程。
方案：强调“支付确认 → 稳定币到账”路径。
- 左侧手机支付确认屏保留
- 中间用清晰箭头/路径连接
- 右侧 USDT token 到账卡
- 加到账状态与金额层
- 构图目标：一眼认出“用 Apple Pay 买 USDT”

### 4. AI Signal
问题：还是抽象 K 线背景，不像 actionable signal。
方案：做成“交易信号面板”。
- 主图是上涨路径和关键节点
- 叠加 entry / target / risk 三段小标签
- 保留信号扫描和节点，但变成辅助层
- 构图目标：一眼认出“AI 给了一个交易判断”

### 5. Macro Breaking
问题：已有冲击感，但事件感仍偏弱。
方案：做成“突发新闻海报”。
- 大面积 breaking 红区
- 地图 + 冲突爆点 + live 标识
- 底部资产联动条（Oil / Gold / BTC / US Stocks）
- 构图目标：一眼认出“中东局势升级，跨资产联动”

### 6. Narrative
问题：更像抽象渐变，不像 narrative trend。
方案：做成“趋势扩散图 + 社区热词”。
- 中心主题节点（OpenClaw）
- 周围关联词 / meme / flows 扩散
- 强调圈层传播与热度外溢
- 构图目标：一眼认出“一个叙事正在扩散变热”

### 7. Promo / Feature
问题：已接近专题卡，但还可以更像内容平台头图。
方案：强化“专题封面感”。
- 左下主标题
- 右上专题小浮层 / card visual 保留
- 增加 editorial column / issue frame / feature rail
- 构图目标：不是广告，而是“值得点开的专题文章”

### 8. Earn
问题：只是绿色收益氛围，不像真实收益机会。
方案：做成“收益产品快照”。
- APY 大数字 + 收益曲线
- 灵活/锁仓标签
- 周期收益条或资金流入感
- 构图目标：一眼认出“这个池子收益不错，可以存”

## System Rules
- 所有卡片保留统一骨架：圆角、内框、光泽、阴影、media headline 位置统一
- 但每张卡必须有 **一个主语义对象**，不能只靠背景和装饰
- 装饰元素只能辅助主语义，不能喧宾夺主
- 图上信息优先回答：**这是什么内容？为什么值得点？**

## Success Criteria
用户快速下滑时：
- 不读详细标题，也能区分出这是 KOL、支付、AI 信号、宏观突发、理财机会
- 不会觉得是一组“同一模板换颜色”的占位图
- 整页仍保持统一系统感
