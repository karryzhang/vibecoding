# Wallet Feed Lab Design

**Goal**
把 `wallet-feed-lab.html` 从“概念型高保真草图”打磨成“高标准、可被真实产品采用的移动端钱包首页实验稿”。标准不是模仿 Apple 的具体界面，而是按其产品完成度要求自己：克制、精确、统一、可信。

## 设计原则
1. **秩序优先于装饰**：先解决对齐、间距、层级，再谈氛围。
2. **克制优先于炫技**：减少廉价渐变、过多发光、无意义装饰。
3. **产品文案优先于概念文案**：页面不解释自己，只帮助用户判断和行动。
4. **系统一致性**：顶部、内容流、底部 follow-up 必须属于同一套产品语言。
5. **信息流节奏**：用户一眼能分辨“现在最重要”“值得继续看”“下一步动作”。

## 本轮重构范围
### 1. Layout / Spacing
- 统一页面水平边界与卡片内边距
- 统一卡片圆角、按钮高度、标题与正文间距
- 收紧 section head、topbar、hero、feed、follow-up 的节奏

### 2. Visual Language
- 降低背景装饰感、减弱发光和过强渐变
- timeline 变成自然的结构元素，而不是外挂装饰
- 让卡片更像成熟产品里的信息模块，而不是设计练习

### 3. Information Hierarchy
- 强化 hero 的主次关系
- 让 signal / story / action 三类卡有一致骨架但不同角色
- 让 follow-up 像真实能力区，而不是尾部总结

### 4. Copy System
- 删除“解释页面/解释设计”的表述
- 改成短、准、直接帮助决策的产品语言
- 缩短标题，减少口号感和设计腔

## 推荐方案
采用“**Apple 标准，不是 Apple 造型**”的中度克制方案：
- 保留实时市场产品应有的张力
- 但所有张力都必须服务于信息判断，而不是视觉表演

## 验收标准
- 截图状态下，页面第一眼就显得更整洁、更安静、更贵
- 任意两张卡片之间的内外边距看起来属于同一系统
- 文案不再像概念说明，而像可上线产品
- timeline、follow-up、hero 不再各说各话
