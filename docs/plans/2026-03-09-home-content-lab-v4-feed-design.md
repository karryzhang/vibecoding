# Home Content Lab V4 设计文档 — 时间优先的统一信息流

**日期：** 2026-03-09
**项目：** VibeCoding
**方向：** 统一单列 feed item，按时间倒序组织首页信息

## 目标
将原本的首页卡片混排实验，重构为更接近真实钱包/交易 App 的首页信息流：
- 时间是最高优先级
- 最新最重要的信息放在最上方
- 所有信息统一进入同一种 feed item 骨架
- 保留轻量钱包首页壳子，但不抢信息流主视觉

## 页面结构
1. 顶部轻壳
   - 头像 / 搜索 / 通知
   - 轻资产摘要：总资产 + 24h 变化
   - 轻操作入口：Trade / Earn / Discover / Watchlist
2. 主内容区
   - 单列 feed
   - 按时间倒序排列
   - 所有信息采用统一视觉结构
3. 底部导航
   - 维持 H5 首页语境

## 统一 feed item 结构
每条内容统一 5 层：
1. 时间
2. 类型标签（运营 / AI / 宏观 / 叙事 / Earn）
3. 标题
4. 一行影响说明
5. CTA 按钮

## 信息类型示例
- 运营热点：人工精选重点内容
- AI 热点：机会型结论，如巨鲸涌入、BTC 短时突破
- 宏观事件：战争/政策/宏观转折及其资产联动
- 热门叙事：OpenClaw 等 narrative pulse
- 理财机会：如 SOL APY 上升至 6.5%

## 设计原则
- 不再使用大卡/小卡/横滑卡等多套语法
- 页面应像高密度但清晰的金融资讯流，而非广告拼贴页
- 类型差异主要通过标签、文案、CTA 和轻量色彩完成
- 强调“可扫读”，移动端用户 1 秒能理解一条信息

## 第一版范围
- 静态 demo
- 中英文双语
- 亮暗主题
- 5 条以上示例 feed item
- 不接真实数据

## 成功标准
- 用户一眼能看出这是时间倒序的信息流首页
- 各种内容类型能自然共存，不显得像广告堆
- 用户能快速判断哪条最新、哪条更值得点开
