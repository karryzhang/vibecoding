AI 基础概念

在安装任何工具之前，先弄清楚这些概念——它们是理解 OpenClaw 如何运作的基石。

🤖

AI（人工智能）

让计算机模拟人类思维的技术。现代 AI 通过海量数据训练，能理解文字、图片和语音。

📚

LLM（大语言模型）

像 GPT-4、Claude、Gemini 这类模型。训练了数千亿文字，能进行对话、写代码、分析问题。

🦾

AI Agent（智能体）

比普通 AI 更进一步 — 能主动调用工具、搜索网络、操作文件，像一个真正的助理。

🔌

API（接口）

你付费购买 AI 模型的「使用权」。OpenAI、Anthropic 等公司提供 API Key 来调用模型。

🌉

Gateway（网关）

连接你的聊天软件和 AI 模型的桥梁。OpenClaw 就是这样一个 Gateway。

🪄

Prompt（提示词）

你发给 AI 的指令。写得越清晰，AI 的输出越准确。「会提问」是使用 AI 的核心技能。

VPN 与网络环境

⚠️

为什么需要 VPN：OpenAI、Anthropic、Google 等主流 AI 服务在中国大陆无法直连。使用 OpenClaw 时，你的服务器或本机需要能访问这些 API。

🔧 两种部署方案

A
本机运行 + VPN
在自己的电脑上安装 OpenClaw，配合 VPN 翻墙。
优点：操作简单，适合个人使用。
缺点：电脑关机 AI 就离线，VPN 稳定性影响体验。
B
海外服务器运行（推荐）
在香港、新加坡等地的云服务器上安装 OpenClaw。
优点：24小时在线，速度快，无需 VPN。
推荐平台：Hetzner（德国/美国，最便宜）、DigitalOcean、Vultr、AWS Lightsail。
选购建议 
新加坡 / 香港节点 = 延迟最低（推荐亚洲用户）
最低配置：1 vCPU + 512MB RAM + 10GB 存储
月费：约 $4–$6 美元（Hetzner 最便宜）

推荐 VPN 工具（本机方案用）

Clash 推荐

主流翻墙工具，规则灵活。可以只让 AI API 请求走代理，其他流量直连。

免费客户端 + 需购买订阅

iOS 最佳选择（$2.99 一次买断）。配合机场使用，稳定流畅。

$2.99（App Store 美区）

$49.99（买断）

OpenClaw 是什么

OpenClaw 是一个开源的 AI Agent Gateway（智能体网关）。它运行在你的设备上，把 AI 模型接入你日常使用的聊天软件。

系统架构图
Telegram / WhatsApp
你的消息入口
→
🦞 OpenClaw Gateway
核心路由引擎
→
Claude / GPT-4
AI 模型 API
你在 Telegram 发消息 → OpenClaw 接收 → 调用 AI → 返回结果

💬

Telegram、WhatsApp、Discord、iMessage 全部接入，统一管理。

🔀

🛠️

Agent 工具

AI 能搜索网络、读写文件、执行代码，不只是聊天。

📱

配对 iOS / Android，让 AI 看到你的屏幕、拍照分析。

🌐

Web 控制台

浏览器打开即可管理会话、配置、监控日志。

⏰

定时任务

设置 AI 定时执行任务：每天早报、自动提醒、定期监控。

安装与配置

💡

前置要求：需要安装 Node.js 24（最低 Node 22）。检查方式：终端运行 node --version。

📦 安装 Node.js

1
macOS 安装 Node（推荐用 nvm）
bash 
# 安装 nvm（Node 版本管理器）
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash

# 重新打开终端后，安装 Node 24
nvm install 24
nvm use 24

# 验证
node --version  # 应显示 v24.x.x
2
Linux / 云服务器安装 Node
bash 
# Ubuntu / Debian
curl -fsSL https://deb.nodesource.com/setup_24.x | sudo -E bash -
sudo apt-get install -y nodejs

# 验证
node --version
3
🪟 Windows 安装指南（WSL2 + Node）
Windows 推荐通过 WSL2（Windows 子系统 Linux）运行 OpenClaw，体验和 Linux 完全一致，且官方强烈推荐。
ℹ️

为什么用 WSL2？OpenClaw 是 Node.js 程序，在 WSL2 的 Linux 环境里运行最稳定，避免 Windows 路径/权限等兼容性问题。

第一步：安装 WSL2
PowerShell（以管理员身份运行） 
# 一键安装 WSL2 + Ubuntu（Windows 10/11）
wsl --install

# 安装完成后重启电脑，然后打开「Ubuntu」应用
# 首次进入设置用户名和密码
💡

如果 wsl --install 报错，先运行：wsl --update，或在「控制面板 → 程序 → 启用或关闭 Windows 功能」里勾选「适用于 Linux 的 Windows 子系统」和「虚拟机平台」。

第二步：在 WSL2 里安装 Node.js
bash（Ubuntu 终端里执行） 
# 安装 nvm
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash

# 重新加载 shell 配置
source ~/.bashrc

# 安装 Node 24
nvm install 24
nvm use 24

# 验证
node --version  # 显示 v24.x.x 即成功
第三步：VPN 代理穿透到 WSL2
WSL2 默认走 Windows 宿主机的网络，但代理不会自动传入。需要手动配置：
bash — 在 WSL2 终端里设置代理 
# 获取 Windows 宿主机 IP（每次启动可能变化）
export HOST_IP=$(cat /etc/resolv.conf | grep nameserver | awk '{print $2}')

# 设置代理（端口改为你的 Clash/VPN 监听端口，一般是 7890）
export http_proxy="http://$HOST_IP:7890"
export https_proxy="http://$HOST_IP:7890"

# 验证代理是否生效
curl -I https://api.telegram.org
💡

把上面的 export 命令加到 ~/.bashrc 末尾，就不用每次手动设置了。Clash 需要开启「允许局域网连接」选项。

推荐终端工具
💻

Windows Terminal

微软官方终端，支持多标签，从 Microsoft Store 免费安装。比 cmd 好用 100 倍。

⚡

VS Code + WSL 扩展

安装 VS Code 后装「WSL」扩展，可以直接在 Windows 里编辑 WSL 内的文件，体验极佳。

接入渠道 — 以 Telegram 为例

Telegram Bot 是最推荐的入门渠道：免费、稳定、功能最完整。下面完整走一遍接入流程。

1
在 Telegram 创建 Bot
打开 Telegram，搜索 @BotFather，点击对话，发送 /newbot。
  • 输入 Bot 名称（如：「我的 AI 助手」）
  • 输入 Bot 用户名（必须以 bot 结尾，如 my_ai_bot）
  • BotFather 返回一串 Token，格式：1234567890:AAXXXXXXXX
  • ⚠️ 保存好这个 Token，不要泄露
2
配置 Token 到 OpenClaw
编辑配置文件 ~/.openclaw/openclaw.json：
json — ~/.openclaw/openclaw.json 
{
  "channels": {
    "telegram": {
      "enabled": true,
      "botToken": "你的Token填这里",
      "dmPolicy": "pairing"
    }
  }
}
💡

dmPolicy: "pairing" 表示第一次有人私信 Bot 时，需要配对验证 — 这样陌生人无法使用你的 AI 助手。

3
重启 Gateway 并配对
bash 
openclaw gateway restart
然后在 Telegram 搜索你的 Bot，发送任意消息。Bot 会返回一个 配对码，在终端执行：
bash 
# 查看待配对列表
openclaw pairing list telegram

# 批准配对（CODE 替换为 Bot 返回的验证码）
openclaw pairing approve telegram CODE
4
🎉 开始聊天！
配对成功后，直接在 Telegram 发消息给 Bot，就能和 AI 对话了。
试试发送：你好，介绍一下你自己
✨

实用命令（发给 Bot 即可）：
/status — 查看当前状态
/model — 切换 AI 模型
/reset — 清空对话历史
/new — 开始新会话

🔒 安全配置（可选但推荐）

json — 白名单配置 
{
  "channels": {
    "telegram": {
      "enabled": true,
      "botToken": "你的Token",
      "dmPolicy": "allowlist",
      "allowFrom": ["你的Telegram数字ID"]
    }
  }
}
ℹ️

如何获取你的 Telegram 用户 ID：启动 Gateway 后私信你的 Bot，然后运行 openclaw logs --follow，查找 from.id 字段即是你的 ID。

接入 AI 模型

OpenClaw 支持几乎所有主流 AI 模型。核心区别：有些需要付费 API Key，有些支持免费 OAuth 授权。

A
Claude（Anthropic）— OAuth 免费登录
Anthropic 官方允许通过 OAuth 使用 Claude.ai 订阅，无需单独购买 API。
bash 
# 打开 Anthropic OAuth 授权
openclaw auth login anthropic

# 按提示在浏览器完成授权
# 授权后自动配置，无需填写 API Key
B
GPT-4（OpenAI）— API Key 方式
前往 platform.openai.com 创建 API Key，然后配置：
json — openclaw.json 
{
  "agents": {
    "defaults": {
      "model": "openai/gpt-4o",
      "auth": {
        "openai": {
          "apiKey": "sk-你的Key"
        }
      }
    }
  }
}
⚠️

API Key 有费用。OpenAI 按 token 计费，日常使用约 $1–5/月。记得在 platform.openai.com 设置用量上限。

C
Gemini（Google）— 免费额度最多
前往 aistudio.google.com 获取免费 API Key（需 VPN）：
json 
{
  "agents": {
    "defaults": {
      "model": "google/gemini-2.0-flash",
      "auth": {
        "google": {
          "apiKey": "AIza你的Key"
        }
      }
    }
  }
}
D
🇨🇳 DeepSeek — 国内直连，无需 VPN
前往 platform.deepseek.com 注册并创建 API Key，国内可直接访问：
json — 兼容 OpenAI 格式 
{
  "agents": {
    "defaults": {
      "model": "openai/deepseek-chat",
      "auth": {
        "openai": {
          "apiKey": "sk-你的DeepSeekKey",
          "baseURL": "https://api.deepseek.com/v1"
        }
      }
    }
  }
}
🐋

DeepSeek 优势：价格极低（约 OpenAI 的 1/30），逻辑推理和代码能力媲美 GPT-4，国内服务器直连无需翻墙。新注册有免费额度。

E
🇨🇳 通义千问 Qwen — 阿里云，中文最强
前往 dashscope.aliyuncs.com 开通服务，获取 API Key：
json — 兼容 OpenAI 格式 
{
  "agents": {
    "defaults": {
      "model": "openai/qwen-plus",
      "auth": {
        "openai": {
          "apiKey": "sk-你的QwenKey",
          "baseURL": "https://dashscope.aliyuncs.com/compatible-mode/v1"
        }
      }
    }
  }
}
✨

Qwen 优势：阿里云出品，中文理解和长文处理极佳，有免费额度。qwen-plus 性价比最高，qwen-max 能力更强。

调试技巧

🚨

最重要的调试工具：实时日志。遇到任何问题，先看日志，90% 的问题都能在日志里找到原因。

查看日志

bash 
# 实时追踪日志（最常用）
openclaw logs --follow

# 查看最近 100 行
openclaw logs --lines 100

# 过滤错误
openclaw logs --follow | grep -i error

常见问题排查

❓
Bot 不回复消息
  • 运行 openclaw gateway status 确认 Gateway 在运行
  • 查看日志是否有 error 或 failed
  • 确认 Telegram API 可访问：curl https://api.telegram.org（需翻墙）
  • 检查 Token 是否填写正确（不要有多余空格）
❓
AI 返回错误 / 无响应
  • 日志查找 401（API Key 无效）或 429（超过限速）
  • API Key 余额是否充足，登录对应平台检查
  • 网络是否能访问 AI 服务：curl https://api.anthropic.com
❓
配置修改不生效
bash 
# 修改配置后必须重启
openclaw gateway restart

# 检查配置文件是否是合法 JSON
cat ~/.openclaw/openclaw.json | python3 -m json.tool
❓
全面诊断
bash 
# OpenClaw 内置诊断工具
openclaw doctor

# 检查渠道状态
openclaw channels status

OpenClaw 本身完全开源免费。你需要自行购买 AI 模型的 API（如 OpenAI 按量付费），或使用 Claude.ai 的订阅（通过 OAuth 授权，无需单独购买 API）。服务器费用如果用海外 VPS 大约每月 $4-6。
对话直接从你的服务器/电脑发送到 AI 提供商（Anthropic/OpenAI），OpenClaw 只是本地路由，不会上传对话到第三方服务器。只要你自己的服务器安全，对话就是私密的。建议开启 Telegram Bot 的白名单，防止陌生人使用。
取决于 OpenClaw 运行在哪里。如果安装在 24 小时开机的服务器上，随时可用。如果安装在个人电脑上，电脑需要开着。推荐用海外 VPS（$4-6/月），一直在线。
可以。在配置文件中配置多个模型的 API Key，然后用 /model 命令在 Telegram 里随时切换。或者配置多个 Agent，不同渠道走不同模型。
同一个会话内会记住（上下文窗口限制内）。要让 AI 记住跨会话的信息，可以在工作区的 MEMORY.md 里写入，或者指示 AI 更新记忆文件。每次新会话发 /new 会清空上下文重新开始。
WhatsApp 接入使用非官方协议，理论上有封号风险。建议用备用号，不要用主要工作号。Telegram Bot 是官方 API，完全安全，推荐优先使用 Telegram。
配置 Brave Search API（免费 2000 次/月）。配置后，AI 会自动在需要最新信息时调用搜索工具。也可以主动说：「帮我搜索一下 xxx 的最新情况」。
运行 cat ~/.openclaw/openclaw.json | python3 -m json.tool 检查 JSON 格式是否合法。如果格式错误，修复后重启 Gateway。也可以运行 openclaw doctor 进行全面诊断。
